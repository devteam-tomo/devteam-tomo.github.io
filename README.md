# TOMO Blog 운영 규칙

## 포스팅 규칙

### 네이밍 규칙
- 파일 네이밍 규칙 : ex) **2019-1-12-1_meeting.md**
    - "2019-1-12-" : 해당 부분은 jekyll에서 포스팅 파일 인식을 위하여 정해진 네이밍 규칙입니다.(해당 날짜 내용으로 실제 포스팅 입력 시간 적용됨)
    ![readme](/images/common/readme_1.png)
    - "1_meeting" 
        - 1 : 해당 월의 몇회차 모임인지 표기합니다.
        - meeting : 회의 : meeting, 스터디 내용 : subject 의 구분을 위함 입니다.
    - ".md"  : 모든 포스팅 파일은 마크다운(.md) 형식으로 업로드 해야합니다.

최종적으로 "2019-1-12-1_meeting.md" 의 포스팅인 경우 다음과 같은 URL로 접근이 가능하게 됩니다.

https://devteam-tomo.github.io/meeting/2019/01/12/1_meeting.html

>막상 url을 확인해보니 "meeting" 이라는 부분이 중복되네요. <br/>
디렉토리에서 분류가 되기 때문에, .md 파일에 'meeting' 부분은 제거해도 될 것 같네요.)


### /_posts
- 해당 경로는 실제 포스팅할 내용을 마크다운(.md)파일 형식으로 작성하는 공간입니다.
- [/_posts/meeting] 경로는 **회의록**을 포스팅 하는 디렉토리입니다.
- [/_posts/subject] 경로는 **스터디 주제**를 포스팅 하는 디렉토리입니다.
    - 첫 주에는 해당 경로에 각자의 '스터디 주제'를 기입하여 업로드합니다.
    - 마지막 주(모임 전)까지 해당 파일에 자신의 공부한 내용을 기입하여 수정합니다.

### /images
- 이미지를 업로드 하고 싶은 경우 해당 경로의 자신의 디렉토리를 생성하여 관리합니다.
    (사용 예 : )
    - /images/bread 경로에, 사용할 이미지를 업로드
